\begin{abstract}
This report presents a structured evaluation of AI model capabilities across three domains where usefulness translates directly to economic value: investment decision-making, marketing strategy, and personalized health optimization. For each domain, we designed a 10-prompt evaluation protocol following established eval methodology (60\% core scenarios, 25\% edge cases, 15\% adversarial tests) with multi-axis scoring rubrics. We tested these prompts against current production models, recorded and fact-checked responses, and identified specific capability gaps. Based on our findings, we propose three novel evaluation frameworks -- FinReason-X, PersuasionBench-X, and LongevityBench -- each designed to measure real-world utility rather than academic knowledge. Our key finding: the gap between ``model knows about this domain'' and ``model is actually useful in this domain'' is significant and measurable, and closing it requires domain-specific evals built by people who work in these fields.
\end{abstract}

\section{Introduction}
% ============================================================

\subsection{Motivation}

Most AI benchmarks test what a model knows. Very few test whether a model is actually useful. This distinction matters because knowledge and usefulness are different things. A model can score well on financial exam questions and still give terrible trading advice. It can write grammatically perfect marketing copy that nobody clicks on. It can recite medical facts and still recommend a dangerous supplement combination.

This project started from a simple question: if I'm going to trust an AI model with decisions that involve real money, real business outcomes, and real health consequences, how do I evaluate whether it's actually good enough?

I picked three domains where I have direct experience and where AI usefulness creates measurable economic value:

\begin{enumerate}
    \item \textbf{Investment Decision-Making} -- I build and run AI-powered trading systems. I've seen firsthand where models help and where they fail when real money is on the line.
    \item \textbf{Marketing Strategy} -- I run an AI agent that manages content on X. It's been a useful experiment in understanding the gap between ``sounds good'' and ``actually converts.''
    \item \textbf{Health Optimization} -- I follow an AI-designed longevity protocol. I've experienced the benefits and also the uncertainty about whether the model's reasoning is actually sound.
\end{enumerate}

\subsection{Approach}

For each domain, I followed a consistent methodology:

\begin{enumerate}
    \item Built a knowledge base on existing benchmarks and evaluation methodology
    \item Designed 10 test prompts (6 core, 2 edge cases, 2 adversarial)
    \item Validated prompt quality on cheaper models first
    \item Ran final evaluations on production models
    \item Scored responses using multi-axis rubrics
    \item Fact-checked all factual claims in responses
    \item Proposed a domain-specific evaluation framework based on findings
\end{enumerate}

\subsection{What This Report Is Not}

This is not an academic benchmark paper. It's a practitioner's evaluation. The prompts come from real tasks I've tried to accomplish with AI models. The scoring rubrics reflect what actually matters when you're using these models for real work, not what looks good on a leaderboard.

\section{Methodology}
% ============================================================

\subsection{Evaluation Design Principles}

Our evaluation protocol follows established best practices in LLM evaluation design:

\begin{itemize}
    \item \textbf{60/25/15 distribution}: 60\% core scenarios that represent everyday use, 25\% edge cases that test reasoning depth, 15\% adversarial prompts that test failure modes
    \item \textbf{Multi-axis scoring}: Each prompt scored on 3-4 specific dimensions rather than a single quality score. A response can be factually accurate but overconfident, or creative but mathematically wrong. Single scores hide these tradeoffs.
    \item \textbf{Defined benchmarks}: Every prompt has a pre-defined ``good response'' and ``bad response'' description. This is what separates an evaluation from a vibe check.
    \item \textbf{Adversarial testing}: Prompts specifically designed to trigger known failure modes (hallucination, false confidence, unsafe recommendations)
\end{itemize}

\subsection{Testing Pipeline}

\begin{enumerate}
    \item \textbf{Prompt validation}: All prompts tested against a cheaper model to catch poorly constructed prompts, ambiguity issues, and confirm the scoring rubric works
    \item \textbf{Final evaluation}: Prompts run against the production model with responses recorded
    \item \textbf{Fact-checking}: All factual claims in responses verified against external sources
    \item \textbf{Scoring}: Each response scored immediately using the rubric
\end{enumerate}
